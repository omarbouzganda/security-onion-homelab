# Contribution Guidelines

Thank you for considering contributing to the Security Onion Homelab project! We appreciate your interest and efforts to make our community better. Here are some guidelines to help you get started:

## How to Contribute

1. **Fork the Repository**: Start by forking the repository to your own GitHub account.
2. **Clone Your Fork**: Clone your forked repository to your local machine.
3. **Create a New Branch**: Create a new branch for your feature or bug fix. Use a descriptive name for your branch.
4. **Make Your Changes**: Implement your feature or bug fix, following our coding standards and best practices.
5. **Commit Your Changes**: Commit your changes with a clear and concise commit message.
6. **Push to Your Fork**: Push your changes to your forked repository on GitHub.
7. **Create a Pull Request**: Navigate to the original repository and create a pull request, describing your changes.

## Code of Conduct

Please note that we have a [Code of Conduct](CODE_OF_CONDUCT.md) in this project. You must adhere to it to maintain a friendly and welcoming environment.

## Reporting Issues

If you encounter any issues or have suggestions for improvements, please open an issue. Include as much detail as possible to help us understand the problem.
